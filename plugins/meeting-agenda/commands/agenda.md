# /agenda - MTGアジェンダ作成コマンド

対話形式でミーティングのアジェンダを作成します。

## 実行手順

### Step 1: MTGについて自由に説明してもらう

まず「どんなMTGのアジェンダを作りますか？自由に教えてください」と質問します。

**ユーザーからの入力例**:
- 「来週の週次定例、1時間で5人参加」
- 「新機能のキックオフMTG」
- 「チームの振り返りをやりたい」
- 「部長への進捗報告会」

ユーザーの説明から以下を読み取ります：
- MTGの種類・目的
- 所要時間（記載があれば）
- 参加人数（記載があれば）

**テンプレートから選びたい場合**:
ユーザーが「テンプレートから選びたい」「どんな種類がある？」と言った場合は、以下を提示します：

1. 定例ミーティング
2. 1on1
3. ブレインストーミング
4. 意思決定会議
5. 振り返り（レトロ）
6. キックオフ
7. 報告会
8. ワークショップ

### Step 2: 不足情報を確認

Step 1で得られなかった情報のみ確認します：
- **所要時間**: 何分のMTGか（例: 30分、1時間）
- **参加人数**: おおよその人数（例: 5人、10人程度）

※すでに説明に含まれていれば確認不要

### Step 3: 出力形式を確認

「出力形式はどれにしますか？（デフォルト: Markdown）」と質問します：

1. **Markdown** - GitHub、ドキュメント向け
2. **Slack** - Slack投稿用
3. **Notion** - Notionページ用
4. **Plain** - メール、チャット向け
5. **HTML** - リッチメール向け

### Step 4: アジェンダを生成

収集した情報を元に、以下を含むアジェンダを生成します：

1. **ヘッダー情報**
   - MTG名
   - 日時（要記入）
   - 所要時間
   - 参加者

2. **アジェンダ本体**
   - 議題一覧（番号付き）
   - 各議題の時間配分
   - 担当者（該当する場合）

3. **事前準備チェックリスト**
   - 参加者が準備すべきこと

4. **備考**（必要に応じて）
   - MTGのグラウンドルール
   - 注意事項

## MTGタイプ別のアジェンダテンプレート

### 定例ミーティング
```
1. 開会・前回アクションの確認（5分）
2. 各自の進捗共有（人数 × 3-5分）
3. 課題・ブロッカーの共有（10分）
4. 相談事項・ディスカッション（残り時間-5分）
5. 次回アクションの確認・クロージング（5分）
```

### 1on1
```
1. アイスブレイク・近況確認（5分）
2. メンバーからの共有事項（15-20分）
   - 最近うまくいっていること
   - 困っていること・相談したいこと
3. マネージャーからの共有（5-10分）
4. キャリア・成長について（5-10分）
5. 次回までのアクション確認（5分）
```

### ブレインストーミング
```
1. 開会・本日のテーマ説明（5分）
2. グラウンドルール確認（2分）
   - 批判しない、質より量、便乗OK
3. アイデア発散フェーズ（全体の40-50%）
4. アイデアの整理・グルーピング（15-20%）
5. 優先度付け・投票（15-20%）
6. 次のアクション決定・クロージング（10%）
```

### 意思決定会議
```
1. 開会・本日の決定事項の確認（5分）
2. 背景・経緯の共有（10分）
3. 選択肢の提示と説明（15-20分）
4. 質疑応答・懸念点の洗い出し（10-15分）
5. 評価基準に基づく議論（15-20分）
6. 意思決定・合意形成（5-10分）
7. 次のアクション・担当者決定（5分）
```

### 振り返り（レトロスペクティブ）
```
1. 開会・振り返り期間の確認（3分）
2. 前回アクションの振り返り（5分）
3. Keep - うまくいったこと（10-15分）
4. Problem - 課題・改善点（10-15分）
5. Try - 次に試すこと（10-15分）
6. アクションアイテム決定（5-10分）
7. クロージング（2分）
```

### キックオフ
```
1. 開会・自己紹介（参加者による）
2. プロジェクト/施策の背景・目的（10-15分）
3. ゴール・成功指標の共有（10分）
4. スコープ・やらないことの確認（10分）
5. 体制・役割分担（10分）
6. スケジュール・マイルストーン（10分）
7. コミュニケーション方法の確認（5分）
8. 質疑応答（10分）
9. 次のアクション・クロージング（5分）
```

### 報告会
```
1. 開会（2分）
2. 報告事項（全体の70-80%）
   - 報告者ごとに時間配分
3. 質疑応答（15-20%）
4. クロージング（3分）
```

### ワークショップ
```
1. 開会・本日のゴール説明（5分）
2. インプット・レクチャー（20-30%）
3. ワーク説明（5分）
4. 個人/グループワーク（40-50%）
5. 成果発表・共有（15-20%）
6. まとめ・振り返り（5-10分）
```

## 出力形式別フォーマット

### Markdown形式
```markdown
## [MTG名]

📅 **日時**: （要記入）
⏱ **所要時間**: XX分
👥 **参加者**: XX名

### アジェンダ

| # | 議題 | 時間 | 担当 |
|---|------|------|------|
| 1 | 議題1 | X分 | - |
| 2 | 議題2 | X分 | - |

### 事前準備
- [ ] 準備項目1
- [ ] 準備項目2
```

### Slack形式
```
:calendar: *[MTG名]*
:clock1: XX分 | :busts_in_silhouette: XX名

*アジェンダ*
1️⃣ 議題1（X分）
2️⃣ 議題2（X分）
3️⃣ 議題3（X分）

*事前準備*
• 準備項目1
• 準備項目2
```

### Notion形式
```markdown
# [MTG名]

> 📅 日時: （要記入）
> ⏱ 所要時間: XX分
> 👥 参加者: XX名

## アジェンダ

- [ ] 議題1（X分）
- [ ] 議題2（X分）
- [ ] 議題3（X分）

---

## 事前準備

- [ ] 準備項目1
- [ ] 準備項目2
```

### Plain形式
```
[MTG名]
日時: （要記入）
所要時間: XX分
参加者: XX名

【アジェンダ】
1. 議題1（X分）
2. 議題2（X分）
3. 議題3（X分）

【事前準備】
- 準備項目1
- 準備項目2
```

### HTML形式
```html
<h2>[MTG名]</h2>
<p>
  <strong>📅 日時:</strong> （要記入）<br>
  <strong>⏱ 所要時間:</strong> XX分<br>
  <strong>👥 参加者:</strong> XX名
</p>
<h3>アジェンダ</h3>
<ol>
  <li>議題1（X分）</li>
  <li>議題2（X分）</li>
</ol>
<h3>事前準備</h3>
<ul>
  <li>準備項目1</li>
  <li>準備項目2</li>
</ul>
```

---

## 注意事項

- 時間配分は目安です。MTGの性質に応じて柔軟に調整してください
- 参加人数が多い場合は、発言時間を考慮して議題を調整します
- 1時間を超えるMTGの場合は、途中休憩を提案してください
