# MTGファシリテーション スキル

このスキルは、効果的なミーティングの設計・運営に関する知識を提供します。

---

## 効果的なMTGの原則

### 1. 目的の明確化
- すべてのMTGには明確なゴールが必要
- 「このMTGで何が決まれば成功か」を事前に定義
- 目的が不明確なMTGは開催しない

### 2. 適切な参加者
- 意思決定に必要な人だけを招待
- 「情報共有だけ」の人はメモ共有で代替可能
- Amazon式: ピザ2枚ルール（8人以下）

### 3. 時間の尊重
- 開始時間と終了時間を厳守
- 各議題にタイムボックスを設定
- 延長は例外、早く終わるのは歓迎

---

## MTGタイプ別のベストプラクティス

### 定例ミーティング

**目的**: 定期的な情報共有と課題の早期発見

**ポイント**:
- アジェンダは固定フォーマットで効率化
- 進捗報告は事前に書面で共有し、MTGでは質疑のみ
- 「報告」より「相談」「議論」に時間を使う
- 形骸化を防ぐため、定期的に必要性を見直す

**アンチパターン**:
- 一人が長時間話し続ける
- 毎回「特にありません」が続く
- 参加者が内職している

### 1on1

**目的**: メンバーの成長支援と信頼関係構築

**ポイント**:
- メンバーが主役、マネージャーは聞き役
- 業務報告ではなく、キャリアや悩みにフォーカス
- 心理的安全性を確保（評価面談とは分離）
- 定期的に開催し、キャンセルしない

**質問例**:
- 「最近、仕事で楽しいと感じることは？」
- 「今、困っていることや不安なことは？」
- 「私にサポートしてほしいことは？」
- 「3ヶ月後、どうなっていたい？」

**アンチパターン**:
- マネージャーが一方的に話す
- 業務進捗の確認だけで終わる
- 形式的なチェックボックス消化

### ブレインストーミング

**目的**: 多様なアイデアを発散させる

**グラウンドルール**:
1. **批判厳禁**: どんなアイデアも否定しない
2. **質より量**: たくさん出すことを優先
3. **便乗歓迎**: 他人のアイデアに乗っかる
4. **自由奔放**: 突飛なアイデアも歓迎

**進め方**:
1. テーマと制約条件を明確にする
2. 個人でアイデア出し（付箋など）
3. 全体で共有・グルーピング
4. 投票や評価で優先度付け

**アンチパターン**:
- 発散フェーズで「それは難しい」と言う
- 声の大きい人のアイデアだけが残る
- 評価基準なしに「なんとなく」選ぶ

### 意思決定会議

**目的**: 合意形成と決定

**事前準備**:
- 選択肢を洗い出す
- 各選択肢のメリット・デメリットを整理
- 評価基準を明確にする
- 必要なデータを揃える

**進め方**:
1. 決定事項と背景を共有
2. 選択肢を提示
3. 質疑応答で懸念を解消
4. 評価基準に基づき議論
5. 決定方法を明示（多数決、コンセンサス、決裁者判断）
6. 決定内容と理由を記録

**アンチパターン**:
- 情報不足のまま決めようとする
- 決定権限者が不在
- 決まったことが覆る

### 振り返り（レトロスペクティブ）

**目的**: チームの継続的改善

**フレームワーク例**:

| 名称 | 内容 |
|------|------|
| KPT | Keep / Problem / Try |
| Start-Stop-Continue | 始める / やめる / 続ける |
| 4Ls | Liked / Learned / Lacked / Longed for |
| Mad-Sad-Glad | 怒り / 悲しみ / 喜び |

**ポイント**:
- 人ではなくプロセスに焦点を当てる
- 「犯人探し」ではなく「仕組みの改善」
- Tryは具体的なアクションに落とす
- 前回のTryの振り返りから始める

**アンチパターン**:
- 毎回同じProblemが出る（改善されていない）
- Tryが抽象的で実行されない
- 特定の人を責める雰囲気

### キックオフ

**目的**: プロジェクト/施策の共通認識を作る

**必須コンテンツ**:
1. **Why**: なぜやるのか（背景・目的）
2. **What**: 何を達成するのか（ゴール・成功指標）
3. **How**: どうやるのか（アプローチ・スケジュール）
4. **Who**: 誰がやるのか（体制・役割）
5. **When**: いつまでにやるのか（マイルストーン）

**ポイント**:
- 質疑応答の時間を十分に確保
- スコープ外（やらないこと）も明確に
- 次のアクションを具体的に決めて終わる

### 報告会

**目的**: 情報の一方向伝達

**ポイント**:
- 事前に資料を配布し、読んできてもらう
- MTGでは質疑と補足説明に集中
- 全員が話す必要はない
- 「読めばわかる」内容は省略

**アンチパターン**:
- 資料を読み上げるだけ
- 質問の時間がない
- 報告のための報告

### ワークショップ

**目的**: 参加型で学びや成果を得る

**設計のポイント**:
- インプットとワークのバランス（7:3 または 5:5）
- 明確なゴールとアウトプット
- 適切なグループサイズ（3-5人）
- 時間管理を厳密に

**ファシリテーターの役割**:
- 場の安全性を作る
- 時間管理
- 議論が脱線したら軌道修正
- 発言の少ない人を巻き込む

## タイムマネジメント

### 時間配分の目安

| MTG時間 | 開会 | 本編 | クロージング |
|---------|------|------|--------------|
| 30分 | 2分 | 25分 | 3分 |
| 60分 | 5分 | 50分 | 5分 |
| 90分 | 5分 | 75分+休憩5分 | 5分 |
| 120分 | 5分 | 100分+休憩10分 | 5分 |

### 時間が足りなくなったら

1. **パーキングロット**: 議題を「駐車場」に置いて後日対応
2. **オフライン対応**: 関係者だけで別途MTGを設定
3. **非同期対応**: Slackやドキュメントで継続議論
4. **延長判断**: 全員の合意を得て最大15分まで

## リモートMTGのコツ

### 準備
- 機材チェック（カメラ、マイク、回線）
- 画面共有する資料を事前に開いておく
- 通知をオフにする

### 進行
- カメラONを推奨（表情が見える）
- リアクション機能を活用
- チャットも併用（発言しにくい人向け）
- 沈黙は「考え中」かもしれないので待つ

### エンゲージメント維持
- 5-10分おきに質問や確認を入れる
- 名指しで意見を求める
- ブレイクアウトルームを活用

## アジェンダ作成のチェックリスト

- [ ] MTGの目的/ゴールは明確か
- [ ] 議題ごとの時間配分は適切か
- [ ] 議題の順番は論理的か（重要なものを前半に）
- [ ] 意思決定が必要な議題は明示されているか
- [ ] 事前準備が必要な場合、明記されているか
- [ ] 参加者は適切か（多すぎ/少なすぎないか）
- [ ] 所要時間は現実的か

---

## 議事録作成のチェックリスト

- [ ] 参加者は正確に記録されているか
- [ ] 決定事項は明確に記載されているか
- [ ] アクションアイテムに担当者と期限があるか
- [ ] 保留事項や継続議題は記録されているか
- [ ] 次回MTGの情報は記載されているか
- [ ] 参加者全員に共有されているか
