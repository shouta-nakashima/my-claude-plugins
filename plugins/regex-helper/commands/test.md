# test - 正規表現テスター

正規表現をテスト文字列に対して実行し、結果を確認するコマンドです。

## 実行時の動作

### ステップ1: 正規表現の入力

```
テストしたい正規表現を入力してください:
例: /\d+/g
```

### ステップ2: テスト文字列の入力

```
テスト文字列を入力してください:
（複数行も可）
```

### ステップ3: 結果の表示

#### マッチした場合

```
✓ マッチしました

マッチ結果:
[0] "12345" (位置: 6-11)
[1] "3"     (位置: 16-17)
[2] "9999"  (位置: 22-26)

マッチ数: 3件
```

#### マッチしなかった場合

```
✗ マッチしませんでした

テスト文字列: "hello world"
正規表現:     /\d+/

ヒント: この文字列には数字が含まれていません
```

### ステップ4: 繰り返しテスト

```
他の文字列もテストしますか？
[続ける / 正規表現を変更 / 終了]
```

## 表示する情報

### 基本情報
- マッチの有無（✓ / ✗）
- マッチした文字列
- マッチ位置（開始-終了）
- マッチ数

### キャプチャグループがある場合

```
正規表現: /(\d{4})-(\d{2})-(\d{2})/
テスト:   "2024-01-15"

✓ マッチしました

フルマッチ: "2024-01-15"
グループ1 ($1): "2024"
グループ2 ($2): "01"
グループ3 ($3): "15"
```

### グローバルフラグ（g）がない場合

最初のマッチのみ表示し、以下の注意を添える:

```
注意: gフラグがないため、最初のマッチのみ表示しています
全てのマッチを見るには /pattern/g を使用してください
```

## 便利機能

### ハイライト表示

テスト文字列内のマッチ箇所を視覚的に示す:

```
テスト文字列:
注文番号: [ORD-12345-JP]、発送日: [2024-01-15]
          ^^^^^^^^^^^^^^           ^^^^^^^^^^
```

### 部分マッチの分析

完全にはマッチしないが、部分的にマッチしている場合にヒントを表示:

```
正規表現: /^\d{3}-\d{4}$/
テスト:   "〒123-4567"

✗ マッチしませんでした

分析:
- "123-4567" の部分はパターンに一致
- 先頭の "〒" が ^ でブロックされています

修正案: /〒?\d{3}-\d{4}$/ で〒を許容できます
```
