# explain - 正規表現の解説

正規表現を分解して、各部分の意味をわかりやすく解説するコマンドです。

## 実行時の動作

### ステップ1: 正規表現の入力

```
解説したい正規表現を入力してください:
```

### ステップ2: 分解と解説

入力された正規表現を以下の形式で解説:

```
正規表現: /^(?=.*[A-Z])(?=.*\d)[A-Za-z\d]{8,}$/

分解:

^               文字列の先頭
(?=.*[A-Z])     先読み: 大文字を1つ以上含むことを確認
(?=.*\d)        先読み: 数字を1つ以上含むことを確認
[A-Za-z\d]      英大文字、英小文字、数字のいずれか
{8,}            直前のパターンが8回以上
$               文字列の末尾

要約:
8文字以上で、大文字と数字を最低1つずつ含む英数字のみの文字列

用途:
パスワードのバリデーション（強度チェック）
```

### 解説のポイント

- 各部分を1行ずつ説明
- 専門用語には補足説明を添える
- 最後に「要約」と「用途」を記載
- 複雑なパターン（先読み、後読み、非貪欲マッチ等）は特に丁寧に説明

### よくあるパターンの解説例

#### メタ文字
- `.` → 任意の1文字（改行を除く）
- `*` → 直前のパターンを0回以上繰り返し
- `+` → 直前のパターンを1回以上繰り返し
- `?` → 直前のパターンを0回または1回
- `^` → 文字列の先頭（または行頭）
- `$` → 文字列の末尾（または行末）

#### 文字クラス
- `\d` → 数字 [0-9]
- `\w` → 単語文字 [A-Za-z0-9_]
- `\s` → 空白文字（スペース、タブ、改行など）
- `\D`, `\W`, `\S` → それぞれの否定

#### グループとアサーション
- `(...)` → キャプチャグループ
- `(?:...)` → 非キャプチャグループ
- `(?=...)` → 肯定先読み
- `(?!...)` → 否定先読み
- `(?<=...)` → 肯定後読み
- `(?<!...)` → 否定後読み

## フラグの解説

フラグがある場合はその意味も解説:

- `g` → グローバル（全てのマッチを検索）
- `i` → 大文字小文字を区別しない
- `m` → 複数行モード（^と$が各行に適用）
- `s` → ドットが改行にもマッチ
- `u` → Unicode対応
- `y` → スティッキー（lastIndexの位置からのみマッチ）
