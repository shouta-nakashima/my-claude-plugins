# Evaluate Design

既存のページやコンポーネントのデザインを評価し、AIっぽさを避けるための具体的な改善提案を行います。

## Instructions

このコマンドが実行されたとき:

1. **distinctive-ui-designスキルを読み込む** - 評価基準となるデザイン原則にアクセス

2. **対象ファイルを特定**:
   - ユーザーに評価対象のファイルパスを質問
   - 複数ファイルの場合はGlobパターンで検索(例: `src/components/**/*.tsx`)
   - または現在開いているファイルを対象にする

3. **デザイン評価を実施**:

   以下の4つの主要領域を**スコアリング**(各10点満点):

   ### A. タイポグラフィ評価 (Typography) - /10点

   **チェック項目**:
   - [ ] Inter/Roboto/Open Sansなどの一般的なフォントを避けているか?
   - [ ] 見出しと本文で異なるフォントを使用しているか?
   - [ ] フォントウェイトに十分なコントラストがあるか?(例: 100 vs 900)
   - [ ] フォントサイズに大きな差(3倍以上)があるか?
   - [ ] 日本語コンテンツの場合、独創的な日本語フォントを使用しているか?

   **スコアリング基準**:
   - 10点: 独創的なフォント + 優れたペアリング + 明確なコントラスト
   - 7-9点: まあまあ独創的だが、改善の余地あり
   - 4-6点: やや一般的だが、いくつか良い要素がある
   - 1-3点: 非常に一般的、AIっぽい
   - 0点: デフォルトフォントのみ使用

   ### B. 配色とテーマ評価 (Color & Theming) - /10点

   **チェック項目**:
   - [ ] 紫のグラデーション、白背景+紫などの一般的な配色を避けているか?
   - [ ] 統一感のあるカラーパレット(5-7色)が定義されているか?
   - [ ] IDE/文化的/自然からインスピレーションを受けた配色か?
   - [ ] WCAG AAコントラスト基準を満たしているか?
   - [ ] CSS custom properties(変数)でテーマ管理されているか?

   **スコアリング基準**:
   - 10点: 統一感のある独創的なテーマ + アクセシビリティ準拠
   - 7-9点: 良い配色だが、やや一般的な要素がある
   - 4-6点: 標準的な配色、いくつか改善点あり
   - 1-3点: 非常に一般的、AIっぽい配色
   - 0点: デフォルト配色のみ

   ### C. 背景効果評価 (Backgrounds) - /10点

   **チェック項目**:
   - [ ] 単色背景を避け、深みがあるか?
   - [ ] レイヤー化グラデーション、パターン、テクスチャを使用しているか?
   - [ ] グラスモーフィズムや幾何学パターンなど高度な技法を使用しているか?
   - [ ] 背景が視覚的階層を作り出しているか?
   - [ ] パフォーマンスを考慮した実装か?(CSS優先、画像最小化)

   **スコアリング基準**:
   - 10点: 複数の高度な背景効果 + 視覚的深み
   - 7-9点: いくつかの効果があるが、もっと深みを追加可能
   - 4-6点: 基本的なグラデーションのみ
   - 1-3点: やや単調な背景
   - 0点: 単色背景のみ

   ### D. アニメーション評価 (Animation) - /10点

   **チェック項目**:
   - [ ] 散発的なマイクロインタラクションを避け、戦略的に配置しているか?
   - [ ] ページロード時のスタッガード表示など高インパクトなアニメーションがあるか?
   - [ ] ホバー/フォーカス状態に意味のある動きがあるか?
   - [ ] `prefers-reduced-motion`に対応しているか?
   - [ ] アニメーション速度が適切(300ms以内)か?

   **スコアリング基準**:
   - 10点: 高インパクト + 戦略的配置 + アクセシビリティ対応
   - 7-9点: 良いアニメーションだが、もっと戦略的に配置可能
   - 4-6点: 基本的なアニメーションのみ
   - 1-3点: 過度または不適切なアニメーション
   - 0点: アニメーションなし

   ### 総合評価スコア: /40点

   - **35-40点**: 🌟 優秀 - 非常に独創的で洗練されたデザイン
   - **28-34点**: ✅ 良好 - 独創的な要素が多い、いくつか改善可能
   - **20-27点**: ⚠️ 改善が必要 - いくつか良い要素があるが、もっと独創性が必要
   - **10-19点**: ❌ 大幅な改善が必要 - かなりAIっぽい、抜本的な見直しが必要
   - **0-9点**: 🚨 全面的な再設計推奨 - 非常にジェネリック

4. **具体的な改善提案を提供**:

   各領域について:
   - **現在の問題点**: 具体的に何がAIっぽいか
   - **推奨される変更**: distinctive-ui-designスキルに基づいた具体的な提案
   - **実装例**: コードスニペットで示す
   - **優先度**: High/Medium/Lowで重要度を示す

5. **改善版コードの生成(オプション)**:

   ユーザーが希望する場合:
   - 評価結果に基づいて改善版のコードを生成
   - 変更箇所をコメントで明示
   - Before/After比較を提供

## Tips

- **客観的な評価**: distinctive-ui-designスキルの基準に厳密に従う
- **建設的なフィードバック**: 問題点だけでなく、良い点も必ず言及する
- **実装可能な提案**: 抽象的でなく、具体的なコードで示す
- **優先順位付け**: すべてを一度に変更するのではなく、段階的な改善を提案
- **文脈を考慮**: プロジェクトの種類や目的に応じた評価を行う

## 注意事項

- アクセシビリティを犠牲にする提案は行わない
- WCAG AA基準を常に考慮する
- パフォーマンスへの影響を警告する
- ブランドアイデンティティがある場合は尊重する
- 既存のコードを全否定せず、良い点を見つける
